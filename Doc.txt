## CODING RULES (ALWAYS FOLLOW)
1. ALL secrets go in .env — NEVER hardcode URLs or API keys
2. ALL ES queries use ES|QL syntax (FROM index | WHERE | STATS)
3. ALL agent calls go through agent_caller.py helper, never inline
4. ALL dollar values formatted by value_formatter.py, never inline
5. EVERY function has a docstring with Args and Returns
6. EVERY ES|QL query tested in Kibana Dev Tools BEFORE coding
7. Use TYPE HINTS on every function signature
8. Use logging module, NEVER print() for production code
9. Handle ALL Elasticsearch exceptions explicitly (never bare except)
10. Index names are ALWAYS lowercase with hyphens: factory-iot-data

## ES|QL STYLE GUIDE
- Always start: FROM index-name-*
- Use WHERE before STATS for filtering
- Use STATS...BY for aggregations
- Alias every computed field: STATS waste = SUM(x) BY category
- Always end with | LIMIT N (max 1000)
- Never use deprecated commands (no SORT after GROUP BY)

## AGENT SYSTEM PROMPT RULES
- Every system prompt has: ROLE, MISSION, TOOLS, OUTPUT FORMAT, CONSTRAINTS
- OUTPUT FORMAT is always valid JSON (never markdown in JSON fields)
- CONSTRAINTS section always includes: never fabricate data, always cite tool
- Agent responses always include: confidence_score (0.0-1.0)

- Indexes:    kebab-case     factory-iot-data, hospital-drugs, nyc-buildings

# data/generate_factory.py
from elasticsearch import Elasticsearch
from elasticsearch.helpers import bulk
from datetime import datetime, timedelta
from dotenv import load_dotenv
import numpy as np, os

load_dotenv()
es = Elasticsearch(os.getenv('ELASTIC_URL'), api_key=os.getenv('ELASTIC_API_KEY'))

MACHINES = ['PRESS-01', 'PRESS-02', 'PRESS-03']
# PRESS-02 is our anomaly machine — runs even during off-shift
ANOMALY_MACHINE = 'PRESS-02'

def generate_docs():
    """Generate 90 days of factory IoT records."""
    docs = []
    for day in range(90):
        ts = datetime.utcnow() - timedelta(days=day)
        hour = day % 24
        shift_active = 8 <= hour <= 16
        for machine in MACHINES:
            if shift_active:
                runtime = int(np.random.normal(460, 20))
                units = int(runtime * np.random.uniform(0.75, 0.85))
            else:
                # Anomaly: PRESS-02 runs heavily off-shift
                if machine == ANOMALY_MACHINE:
                    runtime = int(np.random.normal(380, 30))  # HIGH idle
                else:
                    runtime = int(np.random.normal(45, 15))   # normal idle
                units = 0  # no production off-shift
            docs.append({
                "_index": "factory-iot-data",
                "_source": {
                    "@timestamp": ts.isoformat() + "Z",
                    "machine_id": machine,
                    "shift_active": shift_active,
                    "runtime_minutes": max(0, runtime),
                    "production_units": max(0, units),
                    "cost_per_hour": 112.50
                }
            })
    return docs

docs = generate_docs()
bulk(es, docs)
print(f"✅ Indexed {len(docs)} factory IoT records")

🔍  ALL ES|QL QUERIES — Copy Into Agent Builder Tools

In Kibana → Agent Builder → Tools → Create Tool → Type: ES|QL. Paste each query below.

Tool 1: usage_anomaly — Detect Over-Procurement

Tool Name:  usage_anomaly
Description:  Finds items ordered significantly more than consumed, indicating waste or over-procurement
Parameters:  threshold_pct (float, default 0.25) — minimum over-order ratio to flag

FROM hospital-drugs
| STATS
    total_ordered = SUM(qty_ordered),
    total_used    = SUM(qty_used),
    record_count  = COUNT(*)
  BY drug_name, wing_id
| EVAL
    delta_qty   = total_ordered - total_used,
    waste_ratio = (total_ordered - total_used) / total_ordered
| WHERE waste_ratio > 0.25
| SORT waste_ratio DESC
| LIMIT 20
| KEEP drug_name, wing_id, total_ordered, total_used, delta_qty, waste_ratio

Tool 2: runtime_anomaly — Detect Idle Equipment

Tool Name:  runtime_anomaly
Description:  Finds machines running significant minutes during off-shift hours when no production occurs
Parameters:  min_idle_minutes (int, default 200) — minimum idle runtime to flag per day

FROM factory-iot-data
| WHERE shift_active == false
| STATS
    total_idle_minutes  = SUM(runtime_minutes),
    total_idle_days     = COUNT(*),
    avg_daily_idle      = AVG(runtime_minutes)
  BY machine_id
| EVAL
    idle_hours     = total_idle_minutes / 60,
    cost_per_hour  = 112.50,
    estimated_cost = idle_hours * cost_per_hour
| WHERE avg_daily_idle > 200
| SORT estimated_cost DESC
| LIMIT 10
| KEEP machine_id, total_idle_minutes, idle_hours, avg_daily_idle, estimated_cost

Tool 3: energy_anomaly — Detect Wasted Energy

Tool Name:  energy_anomaly
Description:  Finds buildings consuming high energy despite very low occupancy
Parameters:  max_occupancy_pct (float, default 0.15) — flag when occupancy is below this

FROM nyc-buildings
| STATS
    avg_occupancy     = AVG(occupancy_pct),
    total_energy_kwh  = SUM(energy_kwh),
    avg_daily_energy  = AVG(energy_kwh),
    record_count      = COUNT(*)
  BY building_id, borough
| EVAL
    electricity_rate  = 0.19,
    total_energy_cost = total_energy_kwh * electricity_rate,
    waste_score       = (1 - avg_occupancy) * total_energy_kwh
| WHERE avg_occupancy < 0.15 AND total_energy_kwh > 5000
| SORT total_energy_cost DESC
| LIMIT 15
| KEEP building_id, borough, avg_occupancy, total_energy_kwh, total_energy_cost, waste_score

Tool 4: value_calculator — Look Up Unit Costs

Tool Name:  value_calculator
Description:  Looks up unit cost for a given item key from the pricing reference index
Parameters:  item_key (string) — the item identifier to look up

FROM pricing-reference
| WHERE item_key == ?item_key
| KEEP item_key, item_name, unit_cost_usd, unit_label
| LIMIT 1

# elastic/workflows/action_workflow.yaml
name: ghost-economy-action-workflow
description: >
  Receives verified Ghost Economy findings from the Action Taker agent.
  Sends Slack alert, indexes audit record, updates dashboard.

triggers:
  - type: manual   # Agent 4 calls this via the workflow tool

inputs:
  anomaly_id:    { type: string, required: true }
  entity:        { type: string, required: true }
  dollar_value:  { type: number, required: true }
  category:      { type: string, required: true }
  calculation:   { type: string, required: true }
  confidence:    { type: number, required: true }
  slack_message: { type: string, required: true }

steps:

  # STEP 1: Check if this entity is a known exception
  - id: check_exceptions
    type: elasticsearch.search
    config:
      index: known-exceptions
      query:
        bool:
          must:
            - match: { entity_id: '{{inputs.anomaly_id}}' }
          filter:
            - range:
                start_date: { lte: 'now' }
                end_date:   { gte: 'now' }

  # STEP 2: If exception found, log and stop
  - id: handle_exception
    type: if
    condition: '{{steps.check_exceptions.hits.total.value}} > 0'
    then:
      - id: log_suppressed
        type: elasticsearch.index
        config:
          index: ghost-economy-audit
          document:
            '@timestamp': '{{now}}'
            finding_id: '{{inputs.anomaly_id}}'
            entity: '{{inputs.entity}}'
            dollar_value: '{{inputs.dollar_value}}'
            action_taken: suppressed_exception
    else:
      # STEP 3a: Send Slack notification
      - id: slack_alert
        type: http.post
        config:
          url: '${SLACK_WEBHOOK_URL}'
          headers:
            Content-Type: application/json
          body:
            text: '{{inputs.slack_message}}'
            attachments:
              - color: danger
                fields:
                  - title: Entity
                    value: '{{inputs.entity}}'
                  - title: Dollar Value
                    value: '${{inputs.dollar_value}}'
                  - title: Calculation
                    value: '{{inputs.calculation}}'
                  - title: Confidence
                    value: '{{inputs.confidence}}'
      # STEP 3b: Index audit record
      - id: index_audit
        type: elasticsearch.index
        config:
          index: ghost-economy-audit
          document:
            '@timestamp': '{{now}}'
            finding_id: '{{inputs.anomaly_id}}'
            entity: '{{inputs.entity}}'
            category: '{{inputs.category}}'
            dollar_value: '{{inputs.dollar_value}}'
            calculation: '{{inputs.calculation}}'
            confidence: '{{inputs.confidence}}'

def call_agent(agent_id: str, message: str) -> dict:
    """Call an Elastic Agent Builder agent via Kibana API."""
    url = f'{KIBANA_URL}/api/agent_builder/agents/{agent_id}/chat'
    headers = {
        "Authorization": f"ApiKey {ELASTIC_KEY}",
        "Content-Type": "application/json",
        "kbn-xsrf": "true"
    }
    payload = {
        "messages": [{"role": "user", "content": message}]
    }
    resp = requests.post(url, headers=headers, json=payload, timeout=120)
    resp.raise_for_status()
    raw = resp.json()
    # Extract the assistant's text content
    content = raw.get('message', {}).get('content', '{}')
    return json.loads(content)  # All agent outputs are JSON
